Tradução EN -->PT  https://cardanodocs.com/cardano/addresses/   2018 10 22Endereços em Cardano SLPara enviar e receber valor, endereços são usados em praticamente todas as criptomoedas. Cardano SL suporta três tipos principais de endereços:
	* Endereços de chave pública
	* Endereços script
	* Endereços de redenção

Endereço de chave pública é normal endereço comum a qualquer outra criptomoeda. É um resultado de uma função hash da chave pública. Ler mais sobre endereços chave pública abaixo.Endereços script é usado em transações "Pay to Script Hash" (P2SH). Opera de forma autónoma e age quase como um depósito bancário: envia-se dinheiro para o endereço, no entanto para redimi-lo é necessário satisfazer certas condições, determinadas por um script associado ao endereço. O endereço por si só contém o hash de um script serializado. Ler mais sobre P2SH abaixo.Endereços de redenção é um tipo especial de endereço para redimir ADA. Ler mais sobre endereços de redenção abaixo.Além disso, Cardano SL suporta também xxx endereços. Este tipo de endereço permitir a utilização de endereços customizados no futuro.Como é um endereço?Endereços são XXX bytestrings codificados, como por exemplo:XXXCodificarXXX codificação é o mesmo utilizado por Bitcoin. Utiliza um alfabeto de 58-símbolos para codificar dados, daí o nome. Aqui tem o alfabeto que estamos a utilizar:XXXEvita a utilização de letras e caracteres alfanuméricos que podem parecer ambíguos quando impressos XXX; assim é mais adequando ao utilizador humano que realiza a entrada de dados manualmente, copiando-o de uma fonte visual e também permite a copiar e colar através do duplo clique que normalmente seleciona o string inteiro.Endereços de chave públicaConforme referido na introdução, as carteiras visíveis no interface do utilizador são a representação conveniente do facto de nos pertencer uma chave secreta para gastar o dinheiro nesta carteira em particular. Mas como é que tais gastos são verificados pela rede e como posso receber dinheiro de outras pessoas? A resposta é que em conjunto com a chave secreta que é usado para controlar o valor nas carteiras, a chave pública é gerada. Esta componente pública pode ser conhecida por toda a gente, daí o nome.Um endereço de chave pública contém o hash da sua chave pública.Chaves públicas são também utilizados para verificação de identidade quando se cria uma transação e outros propósitos auxiliares.Pay to Script HashA ideia de Pay to Script Hash (P2SH) é fornecer muita flexibilidade para formular regras complexas para gastar dinheiro. Em vez de enviar a transação para um endereço de chave pública, cria-se um script validador que toma como parâmetro o chamado script de redenção. Para redimir fundos, passamos o script de redenção ao validador para avaliação. Se avaliar como XXX, dinheiro é enviado conforme especificado pelo redentor, caso contrário nada acontece.Para citar Bitcoin Wiki:"A usar P2SH, pode-se enviar bitcoins para um endereço que é seguro em várias formas invulgares sem o conhecimento sobre os detalhes de como a segurança foi montado. O recipiente poderá precisar de assinaturas de várias pessoas para gastar os bitcoins ou a palavra passe poderá ser necessária ou os requisitos poderiam ser completamente únicos.Endereços de redençãoEndereços de redenção são funções PubKey Hash (P2PKH). Tais endereços contêm o hash da chave pública da redenção e esta chave é atualmente a chave pública Ed25519.Outros tipos de endereçosNo futuro, podemos utilizar atualizações ao sistema para introduzir outros tipos de endereço.
